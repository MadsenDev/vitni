Dockside surveillance still (PNG)
================================

The Vitni application expects a 1×1 PNG frame hashed to
`528d4f1e1e4be74f18d29cea6837e8559215577e913dcca6d8b1e38a80e27c1d`.
Binary assets are not shipped inside the repository, but you can re-create
the image locally by decoding the following Base64 payload:

```
iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR4nGNgYAAAAAMAAWgmWQ0AAAAASUVORK5CYII=
```

Run this one-liner from the project root to materialise the PNG in-place:

```bash
printf 'iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR4nGNgYAAAAAMAAWgmWQ0AAAAASUVORK5CYII=' \
  | base64 -d > samples/operation-glass-harbor.vitni/attachments/52/8d/528d4f1e1e4be74f18d29cea6837e8559215577e913dcca6d8b1e38a80e27c1d.png
```

Once generated, the hashed file aligns with the manifest metadata and the
`src-surveillance` record in `db/case.sqlite.sql`.
