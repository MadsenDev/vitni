# Vitni 1.0.0 Checklist

This checklist defines the minimum bar for calling Vitni stable as a `1.0.0` desktop release.

## Product Scope

- [ ] Core product contract is stable: local-first investigation graph, source-backed assertions, timeline, review, and report export.
- [ ] Any intentionally incomplete features are documented as limitations rather than implied as finished workflows.
- [ ] README no longer frames Vitni as primarily "under active development" or lists core workflows as still evolving.
- [ ] User manual covers the primary workflows a new investigator needs to complete a case from project creation through export.

## Release Quality

- [ ] `npm run lint` passes.
- [ ] `npm run test:renderer` passes.
- [ ] `npm run build` passes.
- [ ] Linux package builds successfully from a clean checkout.
- [ ] Windows package builds successfully from a clean checkout.
- [ ] GitHub Release workflow uploads expected Linux and Windows artifacts.
- [ ] Packaged app launches on each supported platform.

## Manual Smoke Test

- [ ] Create a new investigation project.
- [ ] Open the sample project.
- [ ] Add representative people, organization, technology, evidence, location, event, and transaction nodes.
- [ ] Create and edit relationships between nodes.
- [ ] Add sources and source-backed assertions.
- [ ] Review assertions as accepted, disputed, and rejected.
- [ ] Verify conflicts and weakly supported assertions appear in review.
- [ ] Use graph filtering, layout presets, search, and saved views.
- [ ] Verify timeline view shows dated entities correctly.
- [ ] Run local PDF extraction on an attached PDF.
- [ ] Run remote WHOIS, DNS, and IP transforms with consent.
- [ ] Import a CSV into an existing case.
- [ ] Export full, selection, timeline, and person-profile reports.
- [ ] Export with attachments enabled.

## Data And Privacy

- [ ] Project data stays local by default.
- [ ] Remote transforms clearly show what leaves the machine before execution.
- [ ] Local AI setup and disabled states are clear.
- [ ] Cloud AI report generation makes data-sharing scope explicit.
- [ ] Source file hashes and attachment handling behave predictably in packaged builds.

## Documentation

- [ ] README screenshots match the current UI.
- [ ] README setup, sample-case, packaging, and workflow instructions are current.
- [ ] Manual screenshots or descriptions match the current UI where applicable.
- [ ] `CHANGELOG.md` has a complete `1.0.0` section before tagging.
- [ ] Known limitations are listed in release notes.

## Maintenance

- [ ] Browser compatibility data warnings are resolved or accepted as non-blocking.
- [ ] Vite CJS API deprecation warning is resolved or accepted as non-blocking.
- [ ] Renderer bundle-size warning is resolved or accepted as non-blocking.
- [ ] Dependency versions are reviewed for major security or packaging risks.
- [ ] Version number is bumped to `1.0.0` only after the checklist is complete.
