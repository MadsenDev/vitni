# Saved Views

A **saved view** is a named snapshot of the workspace state. It captures: the active workspace (graph/timeline/review), the sidebar tab, which nodes are selected, visible node types, and the camera position on the canvas.

## Saving a View

Click the **View** dropdown in the toolbar → **Save view**. Give it a name. The view is stored in the project.

To update an existing view with the current state, select it from the dropdown and click **Save** (the floppy disk icon).

## Restoring a View

Select the view from the dropdown. The workspace restores to the saved state. If nodes saved in the view have since been deleted, those references are silently dropped.

## Use Cases

- Save a "starting position" view of the full graph to return to after deep dives
- Save a filtered view showing only financial entities for a focused session
- Save a review-in-progress state with specific assertions selected
